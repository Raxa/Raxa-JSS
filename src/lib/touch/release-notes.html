Release Notes for Sencha Touch 2.0.pr4

Release Notes: Release Notes: January 23, 2012
Version Number: 2.0.0.pr4

  • Enhancements/Changes
    • Added 7 new guides, including a 1.x -> 2.x Migration Guide
    • Data refactored:
      • The data packed now makes full use of the config system and we recommend that configs be used with the data package as with the rest of your applications.
      • The performance of the data package has been signifigantly improved.
      • There are some event changes in the Data package and backwards compatibility for the old add, update, and remove events is not fully in place. The old events have severe performance implications and we strongly recommend all developers change to the new addrecords, updaterecord, and removerecords. In addition we have added a new 'refresh event' to supplant the old 'datachanged event' which was fired under too many circumstances.
      • Many issues with LocalStore/WebStore are resolved and this should now work properly. Please note that assosciated local stores are not in place yet.
      • Store paging is back in place and operational.
    • MVC refactored:
      • All MVC classes now consolidated under the Ext.app.* namespace
      • All classes now use the 2.x class and config system
      • Routes can now be defined on Controllers
      • The Router from 1.x has been reinstated
      • Device Profiles are now able to load their own dependencies if activated
      • Added 3 new guides - Intro to apps, Controllers and Device Profiles
      • Kitchen Sink updated to use device profiles, history and deep linking support
    • ComponentView refactored:
      • ComponentView is now fully integrated into DataView and it turned on by using the useComponents: true flag as a configuration option
      • The event signatures for DataView have been changed to handle both Element based items and Component based items. In addition the record is now passed on all the item events.
